Status legend: `[ ]` open · `[~]` in progress · `[x]` done · `[!]` blocked (note why in status file).
Work strictly top-down within a priority band. Acceptance criteria are testable on purpose.

## P0 — Pipeline core (no demo without these)

- [ ] **T-01 Scaffold + CI** — package layout per AGENTS.md, `make check` (ruff + pytest), master seed config.
  *Accept:* `make check` green on empty test suite; `python -m sentinel --version` works.
- [ ] **T-02 Data capture** — script pulling N days of tx history for 1–3 Mantle contracts via RPC into `bench/data/{contract}/raw.jsonl` (schema in BENCHMARK_PROTOCOL §1.2) + SHA-256 manifest.
  *Accept:* snapshot for ≥1 active contract with ≥2,000 txs; rerun is idempotent.
- [ ] **T-03 Tier 2: HDC encoder** — bipolar hypervectors, D=4,096 default; role-filler binding per feature (caller, selector, gas/value/timing buckets); **level encoding for ordinal buckets**; window bundling (W=50 default).
  *Accept:* unit tests — (a) same tx ⇒ identical vector; (b) adjacent gas buckets ⇒ similarity > random pair; (c) window bundle vs prototype similarity > single-tx vs prototype.
- [ ] **T-04 Tier 3: drift signal** — Hamming(window-bundle, prototype) + timing deviation; `normalize()` = rolling median/MAD z-score → squash to [0,1]; `drift = max(...)` exactly as frozen.
  *Accept:* on a stationary synthetic stream drift stays < 0.3; on an injected selector flood drift > 0.7 within 2 windows.
- [ ] **T-05 Tier 1: entropy filter** — per-selector, length-conditioned Shannon entropy baseline; hard threshold; bypasses Tier 4 per freeze.
  *Accept:* normal ABI calldata from snapshot ⇒ 0 alerts; randomized calldata bodies ⇒ alert.
- [ ] **T-06 Tier 4: StaticThresholdDetector** — threshold + hysteresis (k consecutive windows), alert-episode collapsing (10-min).
  *Accept:* unit tests for trigger, hysteresis, episode merge.
- [ ] **T-07 Tier 5: interpreter + Alert JSON** — per-feature similarity probe (unbind roles), name top drifting feature(s); emit alert per `frontend/contracts/alert.schema.json` (extend schema if empty).
  *Accept:* on each injected scenario the interpreter names the injected feature; JSON validates against schema.
- [ ] **T-08 End-to-end replay** — `python -m sentinel replay --snapshot ... [--inject ...]`: transactions → Alert JSON. **This is the MVP goal line.**
  *Accept:* clean replay produces a finite alert list; injected replay produces ≥1 true alert with correct interpreter output.
